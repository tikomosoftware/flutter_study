# 第4回：別のファイルに分ける（ファイル分割）の解説

アプリが大きくなると、1つのファイルに全てのコードを書くのは不可能になります。
ここでは、画面ごとにファイルを分け、それらを連携させる方法を学びます。

## 📍 ファイル構成のルール

今回のレッスンでは、`lib/lessons/lesson_04/` という専用フォルダを作り、そこに3つのファイルを配置しました。

1.  `lesson_04_app.dart`: アプリ全体の入り口（MaterialApp）。
2.  `main_screen.dart`: 最初に表示される「メイン画面」。
3.  `detail_screen.dart`: 遷移先の「詳細画面」。

## 🔗 `import` で繋ぐ方法

他のファイルにあるクラス（ウィジェット）を使うには、ファイルの先頭で `import` を行います。

```dart
// main_screen.dart の例
import 'detail_screen.dart'; // 同じフォルダ内のファイルを読み込む

// これで、detail_screen.dart 内にある "DetailScreen" クラスが使えるようになります。
```

## 🔐 パブリック・プライベートの考え方（Dart独自のルール）

ご質問いただいた「publicなどの宣言は不要か？」という点についてですが、Dartでは非常にシンプルなルールが採用されています。

### 1. 全てはデフォルトで「公開」
他の言語にある `public` キーワードは存在しません。名前の先頭がアルファベット（または `$`）であれば、インポートするだけで自由に使えます。

### 2. 「非公開（Private）」にするには `_` をつける
名前の先頭に **`_`（アンダースコア）** をつけると、そのファイルの外からは見えなくなります。

| 書き方 | 状態 | 他のファイルからのアクセス |
| :--- | :--- | :--- |
| `class MyPage` | 公開 | **可能** |
| `class _InternalPage` | 非公開 | **不可能** |

> [!NOTE]
> これまでは `_Lesson02DialogAppState` のように、Stateクラスに `_` がついていたのを覚えていますか？
> あれは「このStateは外部からいじる必要がない、内部専用のものですよ」ということを示していたのです。
