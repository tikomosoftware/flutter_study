# 第6回：リスト表示（ListView）とモデルクラスの解説

アプリ開発では、「同じような形のデータをたくさん並べて表示する」という場面が非常に多くあります。
今回は、効率的なリスト表示の仕組みと、そのためのデータ管理について学びます。

## 📍 このレッスンのポイント

### 1. モデルクラス（Model Class）
Delphiでいうところの「レコード型」や「クラス」に相当します。
「タイトル」と「完了フラグ」をひとまとめにした `Todo` という独自の型を作ることで、プログラムが非常に扱いやすくなります。

```dart
class Todo {
  final String title;
  bool isDone;
  Todo({required this.title, this.isDone = false});
}
```

### 2. ListView.builder
Flutterでリストを表示するウィジェットはいくつかありますが、最も重要なのがこの `builder` です。

- **効率性**: 1000件のデータがあっても、画面に見えている数件分だけをメモリに読み込んで描画します。
- **itemCount**: リストの総件数を教えます。
- **itemBuilder**: 1行分のデザインを定義します。

### 3. ListTile（リストタイル）
リストの1行を作るための専用パーツです。
- `leading`: 左側のアイコン
- `title`: メインの文字
- `trailing`: 右側のアイコンやボタン
- `onTap`: タップされた時の処理

これらを組み合わせるだけで、標準的な設定アプリのようなデザインがすぐに作れます。

## 💡 Delphiエンジニアへのアドバイス
Delphiの `TListView` では `Items.Add` してデータを追加していくイメージでしたが、Flutterは **「データがこうなっています」と宣言し、更新したくなったら `setState` する** という考え方です。
「データを書き換える」のではなく、「データを書き換えたあと、画面全部を（効率的に）描き直させる」のがFlutter流です。
