# Flutter Web 開発トラブルシューティング

Flutter Webの開発中（ブラウザでの動作確認中）に、「画面が真っ白になる」「コードの変更が反映されない」といった不具合が発生した場合の、基本的な解決手順です。

## 解決手順（3ステップ）

### ステップ1：動いているサーバーを止める
まずは、現在動いている開発サーバーを停止させます。

1. ターミナル（コマンドを打つ黒い画面）をクリックして選択します。
2. キーボードで **`Ctrl + C`** （または `q` キー）を押します。
3. 「Terminate batch job (Y/N)?」と聞かれたら `y` を入力して `Enter` を押します。

### ステップ2：お掃除コマンド（clean）を実行する
Flutterが裏側で自動生成している「一時ファイル」や「キャッシュ」が壊れていることが原因であることが多いです。これを一旦全て削除してまっさらにします。

ターミナルで以下のコマンドを実行します：
```bash
flutter clean
```

### ステップ3：別のポート番号で新しく起動する
ブラウザに残っている古いキャッシュの影響を避けるため、前回とは違う「ポート番号」を指定してサーバーを起動し直します。

ターミナルで以下のコマンドを実行します（※最後の数字を毎回変えるのがコツです）：
```bash
# 例：前回が 8080 だったら 8081 や 8082 などにする
flutter run -d web-server --web-port 8084
```

起動が完了したら、ブラウザで指定したポート番号のURL（例：`http://localhost:8084`）にアクセスして、画面が正常に表示されるか確認してください。

---

### 💡 なぜポート番号を変えるの？
Google Chromeなどのブラウザは、表示を高速化するために「さっきまでそのポート番号（例：8080番）で繋いでいたデータ」を強力に記憶（キャッシュ）しています。
そのため、Flutter側でクリーンアップを行っても、ブラウザが古い記憶を引っ張り出してきてしまい、画面が真っ白なままになることがあります。
ポート番号を変更することで、ブラウザに「これは全く新しい別のサイトですよ！」と認識させ、キャッシュの罠を回避することができます。
