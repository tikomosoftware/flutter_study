# Flutterの画面遷移（スタック）の考え方

Flutterでの画面遷移は、伝統的なWebサイトの「URLを移動する」感覚とは異なり、**「本を積み重ねる（スタック）」**というネイティブアプリ特有の考え方に基づいています。

## 1. Web開発との違い

| 特徴 | 一般的なWebサイト | Flutter (モバイル/デスクトップ) |
| :--- | :--- | :--- |
| **画面の切り替わり** | **ページ遷移**（古いページを捨てて新しいページへ移動） | **スタック移動**（古い画面の上に新しい画面を重ねる） |
| **前の画面** | ブラウザが履歴として持っているが、DOMからは消えている | **スタックの下に実体が残っている**（状態が保持される） |
| **戻る動作** | 新しくページを読み込み直す（ことが多い） | 一番上の画面を取り除くだけ（下の画面がそのまま現れる） |

## 2. スタック（Stack）モデル

Flutterの `Navigator` は、画面を垂直に積み上げる「スタック」というデータ構造で管理しています。

### Push（進む）
新しい画面（Route）をスタックの**一番上に追加**します。
- `Navigator.push(context, route)`
- 下にある画面は見えなくなりますが、メモリ上には存在し続けています。

### Pop（戻る）
スタックの**一番上にある画面を取り除きます**。
- `Navigator.pop(context)`
- 直下にあった画面が再びアクティブになります。

## 3. なぜ「重なる」必要があるのか？

このスタックモデルには、Webにはない強力なメリットがあります。

1.  **状態の保持 (State Persistence)**:
    前の画面が「下にあるだけ」なので、戻った時にスクロール位置や入力内容がそのまま残っています。
2.  **スムーズなアニメーション**:
    2つの画面が同時に存在しているため、重なる際のスライドやフェードといったアニメーションが滑らかに行えます。
3.  **データの受け渡し**:
    `Navigator.pop` をする際に、「結果を返しながら戻る」という処理が非常に簡単に書けます（例：選択画面で選んだ値を前の画面に渡す）。

## 💡 まとめ
Flutterの画面遷移を一言で言うなら、**「1枚ずつウィジェットを積み上げていき、必要なくなったら上から剥がしていく」**という動作です。

Webのように「移動する」のではなく、**「上に重ねる / 下から出す」**という空間的なイメージで捉えると、Flutterの動きがぐっと理解しやすくなります。
